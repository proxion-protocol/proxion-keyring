# Solid Listing Checklist (Kleitikon)

## Core criteria

- Users MUST be able to log in with their WebID and an Identity Provider of
  their choice (Solid-OIDC/WebID-OIDC).
- Data consumed by the app SHOULD be fetched from Solid Pods where possible.
- Data generated by the app MUST be stored in the user’s Solid Pod.
- Interactions between app, pods, and IdP(s) MUST be compliant with Solid
  specifications.

## Excluded categories

Kleitikon MUST NOT be used for harm, unethical uses, hate, malware/data theft,
illegal purposes, or sharing user info without consent. It MUST NOT ship with
known serious security issues.

## App-specific requirements

- Pod-first state: `/kleitikon/` container and subfolders are created on first
  login.
- Clear statement of privacy boundaries and no third-party sharing without
  explicit consent.
- Security features: PT TTL/single-use, PoP, revocation, fail-closed validation.
